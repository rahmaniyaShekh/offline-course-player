# Contributing to Modern Learning Platform

Thank you for your interest in contributing to this project! We welcome contributions from everyone.

## How to Contribute

### Reporting Bugs
1. Check existing issues to avoid duplicates
2. Use the bug report template
3. Include system information (OS, browser, Python version)
4. Provide steps to reproduce the issue
5. Include screenshots or videos if helpful

### Suggesting Features
1. Check existing issues and discussions
2. Use the feature request template
3. Explain the use case and problem being solved
4. Consider implementation complexity
5. Think about backward compatibility

### Code Contributions

#### Getting Started
1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/courses-player.git`
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push to your fork
8. Open a pull request

#### Development Setup
```bash
# Install dependencies
pip install flask

# Run the application
python app.py

# Test your changes at http://localhost:5000
```

#### Code Standards
- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to all functions
- Include comments for complex logic
- Ensure responsive design for frontend changes

#### Before Submitting
- [ ] Test on multiple browsers (Chrome, Firefox, Safari)
- [ ] Test on mobile devices
- [ ] Verify database operations work correctly
- [ ] Check console for JavaScript errors
- [ ] Ensure no broken links or missing files

## Pull Request Process

1. **Update Documentation**: Update README if needed
2. **Add Tests**: Include tests for new functionality
3. **Follow Template**: Use the pull request template
4. **Link Issues**: Reference related issues with "Fixes #123"
5. **Be Patient**: Maintainers will review as soon as possible

## Community Guidelines

- Be respectful and inclusive
- Help others learn and grow
- Provide constructive feedback
- Follow the Code of Conduct
- Credit others for their contributions

## Questions?

- Open a discussion for general questions
- Check existing issues and documentation
- Be specific about what you need help with
