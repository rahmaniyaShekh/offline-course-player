# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes             |
| < Latest| ❌ No              |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability in this project, please follow these steps:

### For Critical Vulnerabilities
1. **Do NOT** open a public issue
2. Email the maintainer directly with details
3. Include "SECURITY" in the subject line
4. Provide detailed information about the vulnerability

### For Non-Critical Vulnerabilities
1. Open a security advisory on GitHub
2. Use the "Report a vulnerability" feature
3. Provide detailed reproduction steps

### What to Include
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if you have one)
- Your contact information

### Response Timeline
- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Fix Development**: Depends on severity
- **Public Disclosure**: After fix is deployed

### Security Best Practices

When using this application:
- Keep dependencies updated
- Use HTTPS in production
- Secure your database files
- Don't expose debug mode in production
- Validate all user inputs
- Use proper file permissions

### Safe Usage Guidelines
- Only run videos from trusted sources
- Be cautious with file uploads
- Regularly backup your progress database
- Monitor for unusual activity in logs

## Contact

For security-related questions or concerns:
- GitHub Security Advisories (preferred)
- Project Issues (for non-sensitive matters)

Thank you for helping keep this project secure!
